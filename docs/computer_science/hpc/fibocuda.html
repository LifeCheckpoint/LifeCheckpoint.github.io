豪俊斐波那契 / 基于CUDA的万亿斐波那契数子串寻踪

初学 CUDA，忍不住上手试验一番并行优化。直入正题，任务目标为：

寻找第 nnn 个斐波那契数 FnF_nFn​，使其低位含有指定的子串

这个问题似乎有一个数学上的算法，暂时不太了解，因此仅做第一次写 CUDA 的练习。初学难免犯错，如有错误请指出。下述算法如果未特殊说明，默认在模意义下讨论。

斐波那契数列的并行递推​

首先是构建搜索过程，如何并行化搜索呢？斐波那契数的计算是串行的，即 Fn+2=Fn+1+FnF_{n+2}=F_{n+1}+F_nFn+2​=Fn+1​+Fn​，如果直接进行并行，每个线程将承受极不平衡的计算量，并且速度也极慢。

那还有什么办法呢？很快就能想到基于递推矩阵的方法。我们有

[Fn+1Fn]=[1110][FnFn−1]\begin{bmatrix} F_{n+1}\\F_{n} \end{bmatrix} =\begin{bmatrix} 1 & 1 \\ 1 & 0 \end{bmatrix} \begin{bmatrix} F_{n} \\ F_{n-1} \end{bmatrix}[Fn+1​Fn​​]=[11​10​][Fn​Fn−1​​]

由此递推，容易得到第 nnn 个斐波那契数的一般表达式

[Fn+1Fn]=[1110]n[F1F0]\begin{bmatrix} F_{n+1} \\ F_{n} \end{bmatrix} =\begin{bmatrix} 1 & 1 \\ 1 & 0 \end{bmatrix}^{n}\begin{bmatrix} F_{1} \\ F_{0} \end{bmatrix}[Fn+1​Fn​​]=[11​10​]n[F1​F0​​]

令 F0=F1=1F_{0}=F_1=1F0​=F1​=1，也就得到 FnF_{n}Fn​ 了。当然，这里还有一个 nnn 次方，最自然的想法就是利用矩阵快速幂。记 F=[1110]F=\begin{bmatrix}1 & 1\\1 & 0\end{bmatrix}F=[11​10​]，先用 O(log⁡n)O(\log n)O(logn) 次矩阵乘法处理出所有 F2kF^{2^k}F2k 的矩阵，接下来再用 O(log⁡n)O(\log n)O(logn) 对 FnF_{n}Fn​ 的每一位进行矩阵乘法，就大功告成啦。

然而还没结束。注意到计算 F2kF^{2^k}F2k 是一个重复性的工作，因此我们可以将这个流程在 CPU 主机端进行预处理，然后剩余的每一位的矩阵乘法，交由 CUDA 进行处理即可。

快速幂算法的并行优化​

继续优化！注意到在 int128 的限制下，每一个线程都要对它们所负责的数字进行 128 次矩阵乘法——还是稍多，经过考虑，我们可以从进制的角度优化一下这个算法。

回顾二进制矩阵幂的过程，主要由两步构成：

  1. 预处理出 F20F^{2^0}F20，F21F^{2^1}F21，F22F^{2^2}F22，...，F2SF^{2^S}F2S 这些幂矩阵，每一个幂矩阵都可以由前一个平方得到，这里一共需要 SSS 次矩阵乘法，储存 SSS 个矩阵
  2. 每个并行线程对于给定数字 NNN，将 NNN 写成 128 位的二进制，如果第 kkk 位为 111，则累乘一个 F2kF^{2^k}F2k，直到最后累乘得到 FN=F2k1⋅F2k2⋅…F^N=F^{2^{k_{1}}}\cdot F^{2^{k_{2}}}\cdot \dotsFN=F2k1​⋅F2k2​⋅…，这里一共需要至多 128 次矩阵乘法

如果我们不拘泥于二进制呢？例如，将基数 222 换成另一个更大的基数 BBB，我们的矩阵幂过程，就会变成这样：

  1. 预处理出 F1⋅B0F^{1\cdot B^0}F1⋅B0，F1⋅B1F^{1\cdot B^1}F1⋅B1，F1⋅B2F^{1\cdot B^2}F1⋅B2，...，F1⋅BSF^{1\cdot B^S}F1⋅BS 这些幂矩阵，每一个幂矩阵都可以由前一个平方得到，这里一共需要 S+1S+1S+1 次矩阵乘法，储存 S+1S+1S+1 个矩阵
  2. 对于每个 F1⋅BkF^{1\cdot B^k}F1⋅Bk 矩阵，继续预处理出 F2⋅BkF^{2\cdot B^k}F2⋅Bk、F3⋅BkF^{3\cdot B^k}F3⋅Bk、...、F(B−1)⋅BkF^{(B-1)\cdot B^k}F(B−1)⋅Bk 这些矩阵，每个矩阵都可以由前一个矩阵乘上 FBkF^{B^k}FBk 得到，这里一共需要 (B−2)×(S+1)(B-2)\times(S+1)(B−2)×(S+1) 次矩阵乘法，储存同样多个矩阵
  3. 每个并行线程对于给定数字 NNN，将 NNN 写成对应位的 BBB 进制，如果第 kkk 位数字为 dkd_{k}dk​，则累乘一个 Fdk⋅BkF^{d_{k}\cdot B^k}Fdk​⋅Bk，直到最后累乘得到 FN=Fdk1⋅Bk1⋅Fdk2⋅Bk2⋅…F^N=F^{d_{k_{1}}\cdot B^{k_{1}}}\cdot F^{d_{k_{2}}\cdot B^{k_{2}}}\cdot \dotsFN=Fdk1​​⋅Bk1​⋅Fdk2​​⋅Bk2​⋅…，这里一共需要至多 log⁡B(2128)\log_{B}(2^{128})logB​(2128) 次矩阵乘法

这里多了一步，也就是将这些幂矩阵的“中间倍数”结果也求了出来。可以看到，当 B>2B>2B>2 时，log⁡B(2128)<128\log_{B}(2^{128})<128logB​(2128)<128，因此这样做确实能减轻内核的计算负担！

那么 BBB 怎么取比较好呢？注意到如果取 BBB 为 2 的幂，则位移起来十分方便快捷，因此可令 B=2bB=2^bB=2b。令 b=8b=8b=8，即 256 进制的矩阵快速幂，我们来看看占用情况：

  1. 预处理第一阶段，需要 S+1S+1S+1 次矩阵乘法
  2. 预处理第二阶段，需要 254×(S+1)254\times(S+1)254×(S+1) 次矩阵乘法
  3. 并行线程，每个线程至多 log⁡256(2128)=16\log_{256}(2^{128})=16log256​(2128)=16 次乘法

非常好！通过将一些计算量下放到主机端预处理，每个 CUDA 线程的计算压力都从 128 次矩阵乘法下降到 16 次，美哉。

主机端

// 计算 Fibonacci 矩阵的 BASE 的幂次方与倍数，结果存储在动态分配的数组中
__host__ Mat2x2** fibo_mat_Bpows() {
    /*
    计算并返回 Fibonacci 矩阵的 BASE 的幂次方与倍数矩阵
    例如，对于 BASE = 4。返回矩阵形如
    I    1 * F^(4 ^ 0)    2 * F^(4 ^ 0)    3 * F^(4 ^ 0)
    I    1 * F^(4 ^ 1)    2 * F^(4 ^ 1)    3 * F^(4 ^ 1)
    I    1 * F^(4 ^ 2)    2 * F^(4 ^ 2)    3 * F^(4 ^ 2)
    ...
    I    1 * F^(4 ^ MAX_MATS_NUM)    2 * F^(4 ^ MAX_MATS_NUM)    3 * F^(4 ^ MAX_MATS_NUM)
    其中 F 是 Fibonacci 矩阵
    */
    // 分配连续内存以便于GPU传输
    Mat2x2 *flat_mats = new Mat2x2[(MAX_MATS_NUM + 1) * BASE];
    Mat2x2 **mats = new Mat2x2*[MAX_MATS_NUM + 1];
    // 设置指针数组指向连续内存的相应位置
    for (int row = 0; row <= MAX_MATS_NUM; row++) {
        mats[row] = &flat_mats[row * BASE];
        mats[row][0] = {1, 0, 0, 1};

        if (row == 0) {
            mats[row][1] = {1, 1, 1, 0}; // F^1 = Fibonacci矩阵
        } else {
            // 计算 F^(BASE^row) = (F^(BASE^(row-1)))^BASE
            Mat2x2 base_power = mats[row - 1][1]; // F^(BASE^(row-1))
            mats[row][1] = base_power;
            for (int i = 1; i < BASE; i++) {
                mats[row][1] = multiply(mats[row][1], base_power, MOD);
            }
        }

        // 计算 col * F^(BASE^row)
        for (int col = 2; col < BASE; col++) {
            mats[row][col] = multiply(mats[row][col - 1], mats[row][1], MOD);
        }
    }

    return mats;
}

子串检查算法​

因为在 GPU 中执行分支会导致分支发散，导致线程空转严重，因此子串检查不能采取一般的 KMP 等算法的思路，而要用顺序性较好、分支较少的检查方式。

这里的检查就较为简省，每次对待检数字进行检查时，都会右移一个十进制位，然后看看除去对应位的 10k10^k10k 后，余数是否就是要找的那个子串。

// 检查 N 是否包含 TARGET_NUMBER 作为十进制子串
__device__ bool contains_decimal_substring(uint128 N) {
    if (N < TARGET_NUMBER) {
        return false;
    }
    uint128 temp_N = N;
    const int MAX_CHECKS = 40 - M; // uint128 的十进制最大有 39 位
    for (int i = 0; i < MAX_CHECKS; ++i) {
        uint128 remainder = temp_N % SUB_NUMBER_10POW;
        if (remainder == TARGET_NUMBER) {
            return true;
        }
        temp_N /= 10;
        if (temp_N < TARGET_NUMBER) {
            break;
        }
    }
    return false;
}
